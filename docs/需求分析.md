## 1. 项目概述

### 1.1 项目背景

PawFinder 是一个面向宠物领养场景的综合平台，采用**微服务架构**设计，连接宠物救助机构与有领养意向的用户，通过严格的审核流程确保领养过程的透明和安全，同时支持在线支付完成领养费用缴纳。

### 1.2 项目目标

- 为流浪宠物提供展示平台，帮助它们找到合适的领养家庭
- 为宠物救助机构提供便捷的宠物管理和领养审核工具
- 为普通用户提供便捷的宠物浏览和领养申请渠道
- 支持在线支付领养费用，确保资金安全
- 采用分布式架构，支持高并发和横向扩展

### 1.3 目标用户

| 用户类型 | 描述 |
|---------|------|
| 游客 | 未登录用户，可浏览宠物和机构信息 |
| 领养人 | 已注册用户，可申请领养宠物、支付费用 |
| 机构用户 | 救助机构工作人员，可发布和管理宠物、审核申请 |
| 管理员 | 系统管理员，负责平台管理和数据统计 |

## 3. 功能需求

### 3.1 用户模块

#### 3.1.1 用户注册与登录

**功能描述**：
- 登录方式：手机号 + 短信验证码
- 自动注册：首次登录时自动创建用户账号
- 角色分配：登录后根据用户角色显示不同功能入口
- 验证码存储：Redis 缓存，5 分钟过期

**业务流程**：
```
用户输入手机号 → 点击获取验证码 → 系统发送验证码（Redis存储）
    ↓
用户输入验证码 → 验证验证码 → 验证通过 → 查询或创建用户 → 生成JWT → 登录成功
```

**输入参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| phone | String | 是 | 手机号 |
| code | String | 是 | 验证码 |

**输出参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| token | String | JWT Token |
| user | Object | 用户信息 |

---

#### 3.1.2 个人信息管理

**功能描述**：
- 修改昵称、头像
- 修改邮箱、地址
- 实名认证（上传身份证）
- 查看申请记录和订单记录

**输入参数**（更新用户）：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 否 | 昵称 |
| email | String | 否 | 邮箱 |
| avatarUrl | String | 否 | 头像 URL |
| bio | String | 否 | 个人简介 |
| address | String | 否 | 地址 |
| idCardNumber | String | 否 | 身份证号 |

### 3.3 领养申请模块

#### 3.3.1 申请提交（用户端）

**功能描述**：
- 填写领养申请表单
- 每个用户对同一宠物只能有一个未取消的申请
- 提交后宠物状态变为 "pending"

**申请表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| petId | String | 是 | 宠物 ID |
| reason | String | 是 | 领养原因 |
| livingCondition | String | 是 | 居住条件描述 |
| experience | String | 是 | 养宠经验 |
| hasOtherPets | Boolean | 否 | 是否有其他宠物 |
| otherPetsDetail | String | 否 | 其他宠物详情 |
| documents | Array | 否 | 证明材料 |
| livingConditionImages | Array | 否 | 居住环境图片 |

---

#### 3.3.2 申请审核（机构用户/管理员）

**功能描述**：
- 查看申请人信息
- 查看申请人提交的领养资料
- 通过/拒绝申请
- 填写审核意见
- **分布式事务**：审核通过时，需同时：
  1. 创建领养记录
  2. 更新宠物状态为 "adopted"
  （使用 Seata AT 模式保证数据一致性）

**审核状态**：
| 状态 | 说明 |
|------|------|
| pending | 待审核 |
| approved | 已通过 |
| rejected | 已拒绝 |
| canceled | 已取消 |

### 3.4 订单支付模块

#### 3.4.1 订单创建

**功能描述**：
- 领养申请审核通过后，用户可创建订单
- 订单号格式：PF + 时间戳 + 6 位随机数
- 订单 24 小时未支付自动取消

**订单字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| orderNo | String | 订单号（唯一） |
| userId | String | 用户 ID |
| applicationId | String | 关联申请 ID |
| petId | String | 关联宠物 ID |
| amount | Decimal | 订单金额 |
| status | String | 状态：pending, paid, canceled, refunded |
| expireAt | DateTime | 过期时间 |

---

#### 3.4.2 支付宝支付

**功能描述**：
- 集成支付宝电脑网站支付
- 生成支付表单，跳转支付宝页面
- 异步回调处理支付结果
- 主动查询支付状态

**支付流程**：
```
用户创建订单 → 点击支付 → 生成支付流水 → 生成支付宝表单 → 跳转支付宝
    ↓
用户完成支付 → 支付宝异步回调 → 系统处理回调 → 更新支付流水 → 更新订单状态
    ↓
（分布式事务，Seata AT 模式）
```

### 7.1 性能需求

| 需求 | 说明 |
|------|------|

| JWT Token | 无状态认证，包含角色信息 |
| 接口鉴权 | 基于角色的访问控制 (RBAC) |
| 支付安全 | 支付宝官方 SDK，签名验证 |
| 敏感数据 | 身份证号等敏感信息加密存储 |
| 操作日志 | 管理员操作日志记录 |
| HTTPS | 生产环境强制 HTTPS |

| 图片优化 | 图片懒加载、WebP 格式 |

## 8. 接口规范

### 8.1 API 路径规范

| 模块 | 路径前缀 | 服务 |
|------|----------|------|
| 用户 | `/api/user/v1` | user-service |
| 宠物 | `/api/pet/v1` | pet-service |
| 领养 | `/api/adoption/v1` | adoption-service |
| 订单 | `/api/order/v1` | order-service |
| 支付 | `/api/payment/v1` | payment-service |
| 搜索 | `/api/search/v1` | search-service |

---

### 8.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": 1680000000000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200 表示成功 |
| message | String | 消息 |
| data | Object | 响应数据 |
| timestamp | Long | 时间戳 |

---

### 8.3 分页响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "current": 1,
    "size": 10,
    "pages": 10,
    "records": [ ... ]
  }
}
```

---

### 8.4 错误码规范

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 4000 | 请求参数错误 |
| 4001 | 业务错误（自定义消息） |
| 4010 | 未登录 |
| 4030 | 无权限 |
| 4040 | 资源不存在 |
| 5000 | 系统错误 |
