## 1. 概述

### 1.1 文档目的

本文档通过**图表为主、代码为辅**的方式，详细展示 PawFinder 宠物领养系统的实现细节。

### 1.2 与设计文档的区别

| 对比项 | 系统设计文档 | 系统实现文档 |
|--------|-------------|-------------|
| **关注点** | What & Why（做什么、为什么） | How（怎么做） |
| **形式** | 文字描述为主 | **图表为主**，代码为辅 |
| **内容** | 架构设计、接口设计 | 时序图、流程图、类图、ER图 |
| **读者** | 架构师、技术决策者 | 开发人员、运维人员 |

### 1.3 技术栈版本

| 层级 | 技术 | 版本 |
|------|------|------|
| **后端框架** | Spring Boot | 3.2.5 |
| **微服务** | Spring Cloud Alibaba | 2023.0.1.0 |
| **数据库** | MySQL | 8.0 |
| **缓存** | Redis | 7.0 |
| **注册中心** | Nacos | 2.3.0 |
| **搜索引擎** | Elasticsearch | 8.12.0 |
| **前端框架** | Next.js | 16.1.1 |
| **UI组件** | shadcn/ui | latest |

## 9. 技术难点攻克

### 9.1 Feign 调用降级问题

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                              Feign 调用降级问题分析                                       │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                         │
│   【问题描述】                                                                           │
│   Feign 调用 pet-service 正常执行，但仍触发降级逻辑                                      │
│                                                                                         │
│   【问题原因】                                                                           │
│                                                                                         │
│   Feign Client 定义:                   Controller 实际路径:                             │
│   ┌───────────────────────────┐        ┌───────────────────────────┐                   │
│   │ @GetMapping(              │        │ @GetMapping(              │                   │
│   │   "/internal/pets/{id}"   │   ≠    │   "/pets/{id}"            │                   │
│   │ )                         │        │ )                         │                   │
│   └───────────────────────────┘        └───────────────────────────┘                   │
│                                                                                         │
│   结果: 请求返回 404，触发 FallbackFactory 降级逻辑                                      │
│                                                                                         │
│   【解决方案】                                                                           │
│                                                                                         │
│   方案1: 修改 Feign Client 路径（推荐）                                                  │
│   ┌───────────────────────────────────────────────────────────────────────────────┐    │
│   │  @FeignClient(name = "pet-service")                                           │    │
│   │  public interface PetClient {                                                 │    │
│   │      @GetMapping("/api/pet/v1/pets/{id}")  // 使用对外接口路径                 │    │
│   │      Result<PetDTO> getPetById(@PathVariable("id") String id);                │    │
│   │  }                                                                            │    │
│   └───────────────────────────────────────────────────────────────────────────────┘    │
│                                                                                         │
│   方案2: 添加内部接口（Controller）                                                      │
│   ┌───────────────────────────────────────────────────────────────────────────────┐    │
│   │  @GetMapping("/internal/pets/{id}")                                           │    │
│   │  public Result<Pet> getPetInternal(@PathVariable String id) { ... }           │    │
│   └───────────────────────────────────────────────────────────────────────────────┘    │
│                                                                                         │
│   【结论】                                                                              │
│   推荐方案1：直接调用对外接口，无需额外维护内部接口                                       │
│                                                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

### 9.2 前端状态枚举大小写问题

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                              前端状态枚举大小写问题                                       │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                         │
│   【问题描述】                                                                           │
│   数据库宠物状态为 AVAILABLE，但前端显示"已领养"                                         │
│                                                                                         │
│   【问题原因】                                                                           │
│                                                                                         │
│   后端返回:                            前端判断:                                        │
│   ┌───────────────────────┐           ┌───────────────────────────────────────┐        │
│   │ status = "AVAILABLE"  │           │ status === "available" ? "可领养" :    │        │
│   │ (大写枚举)            │     ≠     │ status === "pending"   ? "待审核" :    │        │
│   │                       │           │                        "已领养"        │        │
│   └───────────────────────┘           └───────────────────────────────────────┘        │
│                                                                                         │
│   结果: 大小写不匹配，落入默认分支"已领养"                                               │
│                                                                                         │
│   【解决方案】                                                                           │
│                                                                                         │
│   使用 toUpperCase() 统一处理:                                                          │
│   ┌───────────────────────────────────────────────────────────────────────────────┐    │
│   │  const statusConfig = {                                                        │    │
│   │    AVAILABLE: { label: "可领养", color: "green" },                             │    │
│   │    ADOPTED:   { label: "已领养", color: "gray" },                               │    │
│   │    PENDING:   { label: "待审核", color: "yellow" },                             │    │
│   │    UNAVAILABLE: { label: "暂不可领养", color: "red" }                           │    │
│   │  };                                                                            │    │
│   │                                                                                │    │
│   │  const statusInfo = statusConfig[status?.toUpperCase()] || {...}              │    │
│   └───────────────────────────────────────────────────────────────────────────────┘    │
│                                                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```
